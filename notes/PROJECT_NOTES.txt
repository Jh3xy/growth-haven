PLINKO GAME DESIGN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

How it works:
- Player sets bet amount and picks a Risk level: Low / Medium / High
- Player also picks row count: 8, 12, or 16 rows of pegs
- Server generates a random binary path (Left or Right at each peg row)
- Final bucket index = number of Right turns (0 = leftmost, rows = rightmost)
- Each bucket has a multiplier — symmetric, higher risk = wider spread
- Win condition: always resolves (no loss state per se — every bucket pays something, some < 1× = net loss)

Multiplier tables (these go in the RPC as a 2D lookup):
  Low risk:
    8 rows:  [5.6, 2.1, 1.1, 1.0, 0.5, 1.0, 1.1, 2.1, 5.6]
    12 rows: [8.9, 3.0, 1.4, 1.1, 1.0, 0.5, 1.0, 1.1, 1.4, 3.0, 8.9] -- 13 buckets
    16 rows: [15.0, 4.0, 2.0, 1.4, 1.1, 1.0, 0.7, 0.5, 0.7, 1.0, 1.1, 1.4, 2.0, 4.0, 15.0] -- wait, 17 buckets
  Medium risk:
    8 rows:  [13.0, 3.0, 1.3, 0.7, 0.4, 0.7, 1.3, 3.0, 13.0]
    12 rows: [24.0, 5.0, 2.0, 0.9, 0.5, 0.3, 0.5, 0.9, 2.0, 5.0, 24.0] -- wait
  High risk:
    8 rows:  [29.0, 4.0, 1.5, 0.3, 0.2, 0.3, 1.5, 4.0, 29.0]
    12 rows: [46.0, 6.0, 2.0, 0.5, 0.2, 0.2, 0.5, 2.0, 6.0, 46.0] -- wait

NOTE on bucket count: for N rows there are N+1 buckets. You define the exact multiplier arrays in the RPC — design them so they are always symmetric and always house-edge-correct. Use a v_house_edge numeric := 0.01 variable at the top of the RPC (same pattern as all other games — one place to change, flows through all math).

RPC spec — play_plinko:
  Parameters: p_bet_amount numeric, p_rows int (8|12|16), p_risk text ('low'|'medium'|'high')
  SECURITY DEFINER, VOLATILE, search_path = public
  - Declare v_house_edge numeric := 0.01 as first variable
  - Validate: p_bet_amount > 0, p_rows IN (8, 12, 16), p_risk IN ('low','medium','high')
  - Use FOR UPDATE on wallet read
  - Generate path: array of p_rows booleans (true=right, false=left) using random() < 0.5
  - bucket_index = count of true values in path (0 to p_rows)
  - Look up multiplier from hardcoded 2D array by (p_risk, p_rows, bucket_index)
  - won = multiplier >= 1.0
  - payout = p_bet_amount * multiplier
  - profit = payout - p_bet_amount
  - Insert into casino_rounds: game_type='plinko', multiplier=looked_up_multiplier, metadata={path, bucket_index, rows: p_rows, risk: p_risk}
  - Return jsonb: { path (bool[]), bucket_index, multiplier, won, payout, profit, new_balance } or { error } on failure
